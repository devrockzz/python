What are the fetures of oops that java does't support

  • Object Oriented Programing concepts says that the program which we are write should be in the form of objects and classes.
    That means the program is divided into classes and each class can contain variables and methods , if we want to access the variables and methods by using the object of that class we can accsess.

1 : Primitive data types

  • In java there are 8 primitive datatypes.

  • int
  • float
  • char
  • double
  • boolean
  • byte
  • short
  • long
int a = 12;

  • When we declare a variable in our program like int a = 12; , in this statement there is nothing any class and object concepts.But oop says that evrything should be in the form of object and classes.

  • So in this senario java fails to support the feature of oop beacause of using primitive data types.

2 : Multiple Inheritance

  • Multiple inheritance is one of the fetures of object oriented concept , where a class can inherit properties of more than one parent class.

  • In multiple inheritance the problem occurs when methods having same name.This will give an error.

Example - 1

import java.io.*;

class Parent1{

    void print(){

        System.out.println("Inside Parent1 class");

    }

}

class Parent2{

    void print(){

        System.out.println("Inside Parent2 class");

    }

}

class Child extends Parent1 , Parent2{

    public static void main(String arg[]){

        Child c = new Child();

        c.print();

    }

}

Error:

    Child.java:23: error: '{' expected
    class Child extends Parent1 , Parent2{

  • The above program will give an error because multiple inheritance is not supported by java.

  • In this case the compiler wiil confuse because which print() method of which class parent1 or parent2 will executes , so this generates an ambiguity

  • So because of this ambiguity multiple inheritance concept is eliminated from java.
